# Maxim Dokalenko — DJ Promo Site

Static promo website for DJ Maxim Dokalenko. Deployable to GitHub Pages with zero build step.

---

## 📁 Structure

```
/
├── index.html              ← Page shell — tabs, layout
├── style.css               ← All styles + CSS design tokens
├── app.js                  ← All rendering logic
├── update-gallery.js       ← Run locally to update gallery manifest
│
├── data/
│   ├── content.json        ← Bio, tagline, socials, promo card
│   ├── events.json         ← Event calendar
│   └── music.json          ← Embed iframes + local audio files
│
├── gallery/
│   ├── manifest.json       ← Auto-generated by update-gallery.js
│   └── *.jpg / *.png       ← Your photos and flyers
│
└── assets/
    ├── logo.png            ← Header logo (36×36px recommended)
    ├── bio-photo.jpg       ← Bio section portrait
    ├── promo-default.jpg   ← Default promo card image
    └── local-music/
        └── *.mp3           ← Locally hosted audio files
```

---

## 🚀 GitHub Pages Deployment

1. Push this folder to a GitHub repository
2. Go to **Settings → Pages**
3. Set source to **Deploy from branch → main → / (root)**
4. Site will be live at `https://USERNAME.github.io/REPO/`

## ✏️ Content Updates

### Change bio, tagline, or promo card
Edit `data/content.json`

**Key fields:**
- `name` — Your name (used in header, footer, hero)
- `tagline` — Short line shown in hero
- `heroDesc` — One-line description under your name
- `bio` — Array of paragraphs (supports inline HTML like `<strong>`)
- `tags` — Genre/keyword tags shown in bio section
- `socials` — List of `{ "platform": "...", "url": "..." }` objects
- `promo` — The promoted card on the Promo tab:
  - `type` — `"event"`, `"mix"`, `"note"`, or `"photo"`
  - `title`, `subtitle`, `date`, `image`, `link`, `cta`, `note`

### Add or remove events
Edit `data/events.json`

Each event:
```json
{
  "title": "SHOW NAME",
  "venue": "Venue Name",
  "city": "City, Country",
  "date": "2025-06-21",
  "ticketLink": "https://...",
  "note": "Optional short note",
  "past": false
}
```
Set `"past": true` to move to the Past Shows section.
Events with a date in the past are automatically shown as past regardless of this flag.

### Add music embeds
Edit `data/music.json` → `embeds` array.

Each embed:
```json
{
  "platform": "Mixcloud",
  "title": "My Mix Title",
  "iframe": "<iframe ...></iframe>"
}
```
Copy the embed iframe code from Mixcloud/SoundCloud/Bandcamp share options.

### Add local audio files
1. Copy `.mp3` files to `assets/local-music/`
2. Add to `data/music.json` → `localFiles`:
```json
{ "title": "Mix Name", "file": "assets/local-music/your-file.mp3" }
```

## 🖼️ Gallery Updates

1. Copy image files into the `gallery/` folder (any filename)
2. Run: `node update-gallery.js`
3. Commit & push

The site displays images in **random order** (reshuffled on each page load).

Supported formats: `.jpg` `.jpeg` `.png` `.gif` `.webp` `.avif`

## 🎨 Design Customisation

All design tokens are CSS custom properties at the top of `style.css`:

```css
:root {
  --accent: #e8ff00;        /* Main accent colour — change for a whole new vibe */
  --bg:     #0a0a0a;        /* Page background */
  --text:   #f0f0f0;        /* Main text */
  /* ... */
}
```

**To change the colour scheme:** edit `--accent` and related variables.  
**To change fonts:** update the Google Fonts `<link>` in `index.html` and the `--font-*` variables in `style.css`.

## 🧩 Modular Structure

Each section is an independent `render*()` function in `app.js`:

| Function | Section |
|---|---|
| `renderHome()` | Hero / Home tab |
| `renderBio()` | Bio tab |
| `renderGallery()` | Gallery tab + lightbox |
| `renderEvents()` | Events tab |
| `renderMusic()` | Music tab + audio player |
| `renderPromo()` | Promo tab |
| `renderFooter()` | Footer |

To extend a section: edit only its `render*()` function and the corresponding HTML block in `index.html`.
